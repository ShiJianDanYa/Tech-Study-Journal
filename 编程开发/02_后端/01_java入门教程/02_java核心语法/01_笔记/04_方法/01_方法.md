# 2— 核心思想
## 1— 方法
方法：程序中的独立功能，也是最小的执行单元

独立功能：一个方法只负责完成一件具体、单一、不依赖其他无关逻辑的事情 （一个方法只完成一件事）
执行单元：方法是一个打包的整体代码，要么都不执行，要么就从上到下把方法所有代码都执行完毕(要么全部执行,要么全不执行)

优点 提高了程序的<font color="#d83931">复用性</font>和<font color="#d83931">可维护性</font>
## 2— 方法的格式
```java
public static 返回值类型 方法名(参数1,参数2...){
	方法体;
	return 返回值;
}
```
开始推导
方法是一个帮你办事的人
需要让他做事( 需要调用它 )
喊他名字（需要方法名）
告诉他是那两个数(参数列表)
你需要 让他帮你计算 两数之和(方法体)  
最后告诉你结果(返回值)

所以他要做什么
1. 接收数据  （）里面需要参数
2. 计算结果   方法体
3. 返回结果   return 

返回值类型
![[方法_方法格式_2026-04-08_15-50-53.png]]
方便 别人 调用 代码时 直接理解 代码

那两个关键字 暂时理解为固定格式(以后详解)

格式  代码演示
```java
public class 方法 {  
    /*  
    练习：定义一个方法，求两个数的和  
  
    格式：  
    public static 返回值类型 方法名（参数1，参数2...）{  
    方法体；  
    return 返回值；  
    }  
    调用格式：  
    方法名（参数1，参数2...）；  
    注意点：  
    1.方法跟方法之间是平级关系，不能互相嵌套  
    2.方法不会主动运行的，需要被调用才可以  
    3.小括号中的参数需要一一对应（个数，类型）  
    */    public static void main(String[] args){  
        //调用方法  
        System.out.println(getSum(10,20));  
    }  
    public static int getSum(int a,int b){  
        return a + b;  
    }  
}
```

## 3— 方法的注意事项

1. 方法<font color="#d83931">实参形参</font>，<font color="#d83931">返回值和返回值类型</font>要一一对应
2. 方法之间是平行的（包括 main 方法）所以方法的位置是 main 方法外，类的里面
3. 方法需要调用才能执行

方法没有返回值
则方法的返回值类型写 void

## 5— 定义方法的诀窍

需要回答三个问题
 1. 那些代码写到方法中  观察在大段的代码当中， 反复使用的独立功能是什么？反复使用+独立功能

 3. 方法的返回值如何确定 方法的调用处，是否需要这个独立功能的结果继续做其他事情 没有则是void
