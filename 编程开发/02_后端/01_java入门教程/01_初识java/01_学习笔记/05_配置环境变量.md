# 1— 引入问题
如果我想要在CMD的任意目录下，都可以启动某一个软件,该怎么办?
# 2— 核心内容
## 1— 配置环境变量步骤
以前下载的老版本的jdk是没有自动配置的，而且新版的自动配置的也只包含了4个工具而已，所以我们需要删掉已经配置完毕的，再次重新配置Path环境变量。
1. 打开搜索栏,输入环境变量
2. 配置JAVA_HOME
3. 配置path
4. 检测是否安装成功

第四步 检测是否安装成功
按下win+R，
在最下角出现的对话框中输入cmd
![[07_配置环境变量_2026-03-30_18-01-33.png]]
查看JDK版本
再弹出的黑屏窗口中输入命令：java -version
注意1：java后面有一个空格和横杠，然后再输入version
注意2：所有的符号都是英文状态下的
注意3：检查输入是否正确，如正确，按下回车
回车之后，如出现以下信息，则代表安装并设置成功
![[08_配置环境变量_2026-03-30_18-12-01.png]]
解释信息
```java
java version "25.0.1" 2025-10-21 LTS//本次使用 Java 的大版本为25，小版本为0.1，表示把 25 的版本，更新 1 次的小版本，更新时间为：2025 年 10 月 21 日，是LTS 正式版（长期支持版，稳定性拉满，适合学习和生产）
Java(TM) SE Runtime Environment (build 25.0.1+8-LTS-27)//Java 标准版运行环境的构建号为**25.0.1+8-LTS-27**。`27`是一个补丁号，代表这是 JDK25 的第 27 个补丁更新版本，修复了之前版本的 Bug，稳定性更高。
Java HotSpot(TM) 64-Bit Server VM (build 25.0.1+8-LTS-27, mixed mode, sharing)//这表示Java使用的是HotSpot虚拟机，这是一个64位的服务器版虚拟机。构建号与运行时环境相同。mixedmode表示虚拟机运行在混合模式下，即同时使用解释执行和即时编译（JIT）执行Java代码。Asharing表示虚拟机使用了类数据共享（CDS）功能，这可以提高启动速度和减少内存占用。
```
以上三行信息，我们一般只看第一行，知道自己安装的是哪个版本即可，版本只看前面的大版本
## 3— win10的bug
当电脑重启之后，环境变量失效了。表示操作系统不支持自定义的环境变量。
步骤:
* 还是要配置JAVA_HOME给以后的相关软件去使用
* 我们可以把JDK的完整路径配置到PATH当中。E:\develop\JDK\bin
